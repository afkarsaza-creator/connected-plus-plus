--- ðŸ“‹ PANDUAN IMPLEMENTASI & DEPLOYMENT ---

Ikuti langkah-langkah ini untuk menyiapkan dan men-deploy aplikasi cONnected+ Anda sendiri.

**Langkah 1: Pengaturan Backend (Supabase)**

1.  **Buat Proyek Supabase**: Buka [database.new](https://database.new) dan buat proyek baru. Simpan **Project URL** dan **`anon` public key** Anda.
2.  **Perbarui Klien Supabase**:
    *   Buka file `supabaseClient.ts` di proyek Anda.
    *   Ganti nilai `SUPABASE_URL` dan `SUPABASE_ANON_KEY` dengan kredensial dari proyek baru Anda.
3.  **Jalankan Skrip SQL**:
    *   Di dasbor Supabase, navigasikan ke **SQL Editor**.
    *   Buka file `alur_aplikasi.txt` dari proyek ini.
    *   Salin **seluruh isi** file tersebut.
    *   Tempelkan ke SQL Editor dan klik **"RUN"**. Skrip ini akan membuat semua tabel, fungsi, kebijakan keamanan (RLS), dan pemicu yang diperlukan. **Ini adalah langkah krusial.**
4.  **Aktifkan Penyedia Otentikasi**:
    *   Di dasbor Supabase, navigasikan ke **Authentication -> Providers**.
    *   Pastikan penyedia **Email** diaktifkan.

**Langkah 2: Persiapan Frontend**

Aplikasi ini dirancang untuk berjalan langsung di browser tanpa memerlukan proses build lokal (seperti `npm run build`), karena menggunakan impor CDN untuk React. Kode yang ada di folder sudah siap untuk di-deploy.

**Langkah 3: Deployment**

Anda dapat men-deploy folder proyek ini sebagai situs statis menggunakan layanan seperti Vercel, Netlify, atau GitHub Pages.

*Contoh menggunakan Vercel (disarankan karena kemudahannya):*

1.  **Push Kode ke GitHub**: Pastikan semua kode Anda berada di repositori GitHub.
2.  **Hubungkan ke Vercel**:
    *   Daftar/masuk ke [Vercel](https://vercel.com) menggunakan akun GitHub Anda.
    *   Klik **"Add New... -> Project"**.
    *   Impor repositori GitHub Anda.
3.  **Konfigurasi & Deploy**:
    *   Vercel akan secara otomatis mendeteksi bahwa ini adalah proyek statis (tanpa framework). Anda tidak perlu mengubah pengaturan build apa pun.
    *   Klik **"Deploy"**.
4.  **Selesai**: Dalam beberapa saat, Vercel akan memberikan URL publik (misalnya, `nama-proyek-anda.vercel.app`) di mana aplikasi Anda sudah aktif dan siap digunakan.

**Langkah 4: Pengujian Akhir**

1.  Buka URL aplikasi yang telah di-deploy.
2.  Coba buat akun baru. Periksa email Anda untuk tautan verifikasi dari Supabase.
3.  Setelah verifikasi, coba masuk.
4.  Jika Anda dapat masuk dan melihat layar daftar obrolan (meskipun kosong), berarti frontend berhasil terhubung ke backend Supabase Anda. Aplikasi siap digunakan.
